# Bewertungsframework
## Data Visualization & Storytelling - HsH

**Zweck:** Transparente und faire Bewertung der Studierendenleistungen

---

## 📊 Bewertungsübersicht

### Gesamtstruktur
- **Wöchentliche Hausaufgaben:** 60% (9 Aufgaben à 6-7%)
- **Abschlussprojekt:** 40%

### Notenskala (Deutschland)
- 1,0 - 1,3: Sehr gut
- 1,7 - 2,3: Gut
- 2,7 - 3,3: Befriedigend
- 3,7 - 4,0: Ausreichend
- 5,0: Nicht bestanden

### Bestehensgrenze
- **Mindestpunktzahl:** 50% (50 von 100 Punkten)
- **Hausaufgaben:** Mindestens 5 von 9 abgegeben
- **Abschlussprojekt:** Pflicht (mindestens 40% der Projektpunkte)

### Allgemeine Bewertungskriterien

#### 1. Technische Umsetzung (40 Punkte)

**Exzellent (36-40 Punkte):**
- Code funktioniert fehlerfrei
- Effiziente Implementierung
- Bibliotheken korrekt verwendet
- Keine Warnungen oder Fehler
- Best Practices befolgt

**Gut (30-35 Punkte):**
- Code funktioniert mit kleinen Fehlern
- Grundsätzlich korrekte Implementierung
- Bibliotheken meist korrekt verwendet
- Wenige Warnungen

**Befriedigend (24-29 Punkte):**
- Code funktioniert teilweise
- Einige Fehler vorhanden
- Bibliotheken nicht optimal verwendet
- Mehrere Warnungen

**Ausreichend (20-23 Punkte):**
- Code funktioniert mit Einschränkungen
- Viele Fehler
- Bibliotheken falsch verwendet
- Viele Warnungen

**Nicht bestanden (<20 Punkte):**
- Code funktioniert nicht
- Grundlegende Fehler
- Bibliotheken nicht verstanden

#### 2. Visuelle Gestaltung (30 Punkte)

**Exzellent (27-30 Punkte):**
- Professionelles Styling
- Tufte-Prinzipien angewendet
- Hohe Data-Ink Ratio
- Geeignete Plot-Typen
- Ästhetisch ansprechend
- Konsistente Gestaltung

**Gut (24-26 Punkte):**
- Gutes Styling
- Tufte-Prinzipien größtenteils angewendet
- Gute Data-Ink Ratio
- Meist geeignete Plot-Typen
- Ansprechend

**Befriedigend (18-23 Punkte):**
- Grundlegendes Styling
- Tufte-Prinzipien teilweise angewendet
- Mittlere Data-Ink Ratio
- Teilweise ungeeignete Plot-Typen

**Ausreichend (15-17 Punkte):**
- Minimales Styling
- Tufte-Prinzipien kaum angewendet
- Niedrige Data-Ink Ratio
- Oft ungeeignete Plot-Typen

**Nicht bestanden (<15 Punkte):**
- Kein Styling
- Tufte-Prinzipien ignoriert
- Chartjunk vorhanden
- Falsche Plot-Typen

#### 3. Dokumentation (20 Punkte)

**Exzellent (18-20 Punkte):**
- Code vollständig kommentiert
- Markdown-Zellen mit Erklärungen
- Interpretation der Ergebnisse
- Klare Struktur
- Fehlerbehandlung dokumentiert

**Gut (16-17 Punkte):**
- Code gut kommentiert
- Markdown-Zellen vorhanden
- Interpretation vorhanden
- Gute Struktur

**Befriedigend (12-15 Punkte):**
- Code teilweise kommentiert
- Wenige Markdown-Zellen
- Minimale Interpretation
- Grundlegende Struktur

**Ausreichend (10-11 Punkte):**
- Code kaum kommentiert
- Kaum Markdown-Zellen
- Keine Interpretation
- Unstrukturiert

**Nicht bestanden (<10 Punkte):**
- Keine Kommentare
- Keine Markdown-Zellen
- Keine Dokumentation

#### 4. Kreativität (10 Punkte)

**Exzellent (9-10 Punkte):**
- Eigene Ideen eingebracht
- Über Anforderungen hinaus
- Innovative Lösungen
- Zusätzliche Features

**Gut (7-8 Punkte):**
- Einige eigene Ideen
- Anforderungen erfüllt und erweitert
- Gute Lösungen

**Befriedigend (5-6 Punkte):**
- Wenige eigene Ideen
- Anforderungen erfüllt
- Standard-Lösungen

**Ausreichend (4 Punkte):**
- Keine eigenen Ideen
- Nur Mindestanforderungen
- Kopierte Lösungen

**Nicht bestanden (<4 Punkte):**
- Plagiat
- Keine eigene Arbeit

### Bonus-Punkte (max. +10 Punkte)

Können zusätzlich vergeben werden für:
- Besonders innovative Lösungen (+5)
- Hilfe für andere Studierende (+3)
- Zusätzliche Features (+2)
- Exzellente Dokumentation (+2)

**Wichtig:** Bonus-Punkte können die Gesamtpunktzahl über 100 bringen, aber die Hausaufgabe zählt maximal als 100 Punkte zur Gesamtnote.

## 🎓 Abschlussprojekt (40%)

### Übersicht

**Punkte:** 400 (40% der Gesamtnote)  
**Dauer:** 2 Wochen  
**Format:** Jupyter Notebook + Präsentation (15 Min)

### Anforderungen

#### Pflicht-Komponenten
1. **Datensatz:** Eigener oder Kaggle (mindestens 1000 Zeilen)
2. **Visualisierungen:** Mindestens 5-7 verschiedene
3. **Interaktivität:** Dashboard (Streamlit, Dash, oder ähnlich)
4. **Storytelling:** Klare Narrative
5. **Barrierefreiheit:** WCAG AA konform
6. **Dokumentation:** Vollständig
7. **Präsentation:** 15 Minuten + 5 Min Q&A

#### Optional (Bonus)
- Deployment (Streamlit Cloud, Heroku)
- Mehrere Datenquellen
- ML-Integration
- Custom Visualisierungen
- Mehrsprachigkeit

### Bewertungskriterien

#### 1. Technische Umsetzung (160 Punkte = 40%)

**Code-Qualität (80 Punkte):**
- Funktionsfähigkeit (30 Punkte)
- Effizienz (20 Punkte)
- Best Practices (15 Punkte)
- Fehlerbehandlung (15 Punkte)

**Komplexität (40 Punkte):**

- Datenverarbeitung (10 Punkte)
- Interaktivität (10 Punkte)
- Integration verschiedener Tools (5 Punkte)

**Deployment (40 Punkte):**
- Dashboard funktioniert (20 Punkte)

- Performance (10 Punkte)

#### 2. Visuelle Gestaltung (120 Punkte = 30%)

## 📊 Gesamtbewertung

### Berechnung

```
Gesamtpunkte = (Hausaufgaben × 0,6) + (Abschlussprojekt × 0,4)

Beispiel:
Hausaufgaben: 750/900 Punkte = 83,3%
Abschlussprojekt: 340/400 Punkte = 85%

Gesamtpunkte = (750/900 × 60) + (340/400 × 40)
             = 50 + 34
             = 84 Punkte

Note: 1,7 (Gut)
```

## 🔄 Feedback-Prozess

### Hausaufgaben

**Zeitrahmen:** Innerhalb von 1 Woche nach Abgabe

**Format:**
- Punktzahl pro Kriterium
- Schriftliches Feedback (2-3 Absätze)
- Verbesserungsvorschläge
- Positive Aspekte hervorheben

**Beispiel-Feedback:**
```
Hausaufgabe Block 1: 85/100 Punkte

Technische Umsetzung (35/40):
Dein Code funktioniert einwandfrei und ist gut strukturiert. 
Die Verwendung von Matplotlib ist korrekt. Kleine Verbesserung: 
Verwende plt.tight_layout() für bessere Abstände.

Visuelle Gestaltung (26/30):
Sehr schöne Plots mit guter Farbwahl. Die Titel und Labels 
sind klar. Verbesserung: Grid könnte transparenter sein (alpha=0.3).

Dokumentation (18/20):
Exzellente Markdown-Zellen mit klaren Erklärungen. Code ist 
gut kommentiert. Interpretation der Ergebnisse ist vorhanden.

Kreativität (6/10):
Gute Umsetzung der Anforderungen. Für mehr Punkte: Eigene 
Ideen einbringen, z.B. zusätzliche Visualisierungen.

Gesamt: Sehr gute Arbeit! Weiter so!
```

### Abschlussprojekt

**Zeitrahmen:** Innerhalb von 2 Wochen nach Präsentation

**Format:**
- Detaillierte Punktzahl pro Kriterium
- Ausführliches schriftliches Feedback (1-2 Seiten)
- Stärken und Schwächen
- Verbesserungsvorschläge für Portfolio

## 🎯 Häufige Fragen (FAQ)

### Hausaufgaben

**F: Kann ich eine Hausaufgabe nachreichen?**  
A: Ja, bis zu 1 Woche nach Deadline mit 10% Punktabzug.

**F: Was passiert, wenn ich eine Hausaufgabe nicht abgebe?**  
A: 0 Punkte für diese Hausaufgabe. Mindestens 5 von 9 müssen abgegeben werden.

**F: Kann ich Hausaufgaben in Gruppen machen?**  
A: Nein, Hausaufgaben sind Einzelarbeit. Diskussion ist erlaubt, aber Code muss eigen sein.

**F: Welche Programmiersprache muss ich verwenden?**  
A: Python ist Pflicht für Hausaufgaben.

**F: Kann ich andere Bibliotheken verwenden?**  
A: Ja, aber die im Block gelehrten Bibliotheken müssen verwendet werden.

### Abschlussprojekt

**F: Kann ich das Abschlussprojekt in einer Gruppe machen?**  
A: Ja, maximal 2 Personen. Beide präsentieren und werden individuell bewertet.

**F: Muss ich mein Dashboard deployen?**  
A: Nein, aber es gibt Bonus-Punkte dafür.

**F: Kann ich ein Projekt aus meiner Arbeit verwenden?**  
A: Ja, wenn du die Rechte hast und es die Anforderungen erfüllt.

**F: Wie lang sollte meine Präsentation sein?**  
A: Exakt 15 Minuten. Zeitüberschreitung führt zu Punktabzug.

**F: Muss ich Power BI verwenden?**  
A: Nein, du kannst jedes Tool verwenden, das im Kurs behandelt wurde.

### Bewertung

**F: Wie wird die Gesamtnote berechnet?**  
A: 60% Hausaufgaben + 40% Abschlussprojekt.

**F: Was ist die Bestehensgrenze?**  
A: 50% der Gesamtpunkte (50 von 100).

**F: Kann ich eine Hausaufgabe wiederholen?**  
A: Nein, aber du kannst Feedback nutzen für die nächste Hausaufgabe.

**F: Wann bekomme ich Feedback?**  
A: Hausaufgaben: Innerhalb 1 Woche. Abschlussprojekt: Innerhalb 2 Wochen.
